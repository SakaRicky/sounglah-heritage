# S023.6 - Admin Lesson Items Builder Screen

## Epic

Epic 5 - Lessons and Learner Experience (parent: [S023](./00-overview.md))

## Status

Done

## Goal

Provide a curriculum-builder view: ordered list of lesson items for one lesson, with add, edit, reorder, and delete — not a flat global table.

## Dependencies

- S023.4 (lesson exists).
- S023.5 (items API).

## Route

```text
/admin/content/lessons/:lessonId/items
```

## Layout

- **Header:** Lesson title + link back to edit lesson.
- **Meta line:** Difficulty, estimated time, status badge.
- **Primary action:** Add Item → new item form route.
- **Body:** Vertical ordered list (cards), not a wide spreadsheet.

### Each item card

- Order number (1, 2, 3…).
- Type badge (`VOCABULARY`, `PHRASE`, etc.).
- Title.
- Linked concept key/title (or “No concept” for cultural note).
- Active / inactive indicator.
- **Actions:** Edit, Move up, Move down, Delete (confirm dialog).

Empty state: “No steps yet — add the first screen your child will see.”

## Behavior

- Move up/down calls reorder API (`PATCH .../reorder` with `direction`).
- Delete removes item (confirm dialog); backend compacts `order_index` for remaining items in the lesson.
- Inactive items visually muted; still editable.

## Sidebar / navigation (S023.6)

- **Remove** standalone “Lesson Items” link from `AdminSidebar.tsx`.
- Replace `content/lesson-items` placeholder route with redirect to `/admin/content/lessons`.
- Entry point: **Manage Items** action on each lesson row in S023.3.

## Out Of Scope

- Item type form fields (S023.7).
- Drag-and-drop.
- Inline edit of all fields.

## Acceptance Criteria

- Builder loads items in `order_index` order.
- Reorder persists after refresh.
- Add Item navigates to create form.
- Matches Sounglah admin visual system (cream cards, green CTAs).
- i18n EN/FR.
- Lint/build pass.
