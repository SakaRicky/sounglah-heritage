# S023.8 - Public Lessons API + Listing Screen

## Epic

Epic 5 - Lessons and Learner Experience (parent: [S023](./00-overview.md))

## Status

Done

## Goal

Ship anonymous public lesson read APIs and the learner catalog at `/lessons` — no signup, no saved progress.

## Dependencies

- S023.2 (admin lessons exist; public layer reads same tables).
- At least one published lesson for manual QA (full seed in S023.15).

## Backend (required in this slice)

Register **`lessons_bp`** in `create_app` at **`/api/lessons`** — no `@require_admin`.

Add **`backend/app/services/lesson_public_service.py`**:

- `list_published_lessons()` — ordered by `order_index`, title.
- `get_published_lesson_by_slug(slug)` — 404 if missing or not published.
- `build_concept_payload(concept_id)` — resolver rules in [00-overview.md § Public Play Payload Contract](./00-overview.md#public-play-payload-contract).

### Endpoints

```text
GET /api/lessons
GET /api/lessons/:slug
```

| Endpoint | Behavior |
| --- | --- |
| `GET /api/lessons` | Published lessons only; list shape from overview |
| `GET /api/lessons/:slug` | Full detail + active items + `conceptPayload`; **404** for draft/archived/unknown |

Both endpoints are **public** (no auth header). This is intentional for anonymous MVP.

### Tests

- Draft lesson excluded from list and returns 404 on detail.
- Published lesson returns ordered active items only.
- `conceptPayload` resolves EN/FR/MED texts and Médumba audio when present.

## Frontend

### Route

```text
/lessons
```

Use `PublicLayout` from E02. Add route in `frontend/src/app/router.tsx`.

### API client

- `frontend/src/features/lessons/api/publicLessons.ts` (or under `features/lessons/`).
- Types matching public JSON (camelCase).

### UI Contract

- **Header:** Warm title — e.g. “Start learning Médumba” (i18n EN/FR).
- **Subcopy:** Family-centered one-liner.
- **Grid/list of lesson cards:**
  - Cover image.
  - Title.
  - Short description.
  - Difficulty label.
  - Estimated time (“~5 min”).
  - **Open** button → `/lessons/:slug`.

### States

- Empty: friendly message when no published lessons.
- Loading and error states.

## Out Of Scope

- Learner accounts, signup, login.
- Progress / “continue where you left off” (landing page decorative progress stays fake until accounts ship).
- Auth-gated lessons.

## Acceptance Criteria

- `lessons_bp` registered; both GET endpoints work without auth.
- Draft lessons never appear on `/lessons`.
- Cards match Sounglah visual system.
- Mobile-friendly card layout.
- i18n EN/FR.
- Backend tests + frontend lint/build pass.
